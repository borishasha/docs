개발 환경 설정 ​

이 문서는 AxpFramework 프로젝트의 개발 환경을 설정하는 방법을 안내합니다.

1. 프로젝트 클론 및 준비 ​

1.1 프로젝트 다운로드 ​

bash
# Git 저장소에서 클론 (예시)
git clone <repository-url>
cd AxpFramework

1.2 프로젝트 구조 확인 ​

AxpFramework/
├── pom.xml                    # Maven 설정 파일
├── src/
│   ├── main/
│   │   ├── java/             # Java 소스 코드
│   │   └── resources/        # 리소스 파일
│   └── test/                 # 테스트 코드
└── mvnw                       # Maven Wrapper

2. 데이터베이스 설정 ​

2.1 데이터베이스 선택 ​

프로젝트는 다음 데이터베이스 중 하나를 사용할 수 있습니다:

  • Tibero6 (기본값)
  • Oracle
  • PostgreSQL

2.2 설정 파일 수정 ​

src/main/resources/config/application-dev.yaml 파일을 열어 데이터베이스 연결 정보를 수정합니다.

Tibero 설정 예시 ​

yaml
app:
  datasource:
    type: tibero

spring:
  datasource:
    tibero:
      jdbc-url: jdbc:tibero:thin:@localhost:9629:INTDB
      driver-class-name: com.tmax.tibero.jdbc.TbDriver
      username: your_username
      password: your_password

Oracle 설정 예시 ​

yaml
app:
  datasource:
    type: oracle

spring:
  datasource:
    oracle:
      jdbc-url: jdbc:oracle:thin:@localhost:1521:XE
      driver-class-name: oracle.jdbc.OracleDriver
      username: your_username
      password: your_password

PostgreSQL 설정 예시 ​

yaml
app:
  datasource:
    type: postgres

spring:
  datasource:
    postgres:
      jdbc-url: jdbc:postgresql://localhost:5432/your_database
      username: your_username
      password: your_password
      driver-class-name: org.postgresql.Driver

2.3 Tibero JDBC 드라이버 확인 ​

Tibero를 사용하는 경우, src/main/resources/lib/tibero6-jdbc.jar 파일이 존재하는지 확인합니다.

3. Redis 설정 ​

3.1 Redis 설치 및 실행 ​

Windows ​

bash
# Redis 다운로드 및 설치
# https://github.com/microsoftarchive/redis/releases

# Redis 실행
redis-server

Linux/Mac ​

bash
# Redis 설치 (Ubuntu/Debian)
sudo apt-get install redis-server

# Redis 실행
redis-server

3.2 Redis 설정 파일 수정 ​

src/main/resources/config/application-dev.yaml 파일에서 Redis 연결 정보를 수정합니다.

yaml
redis:
  datasource:
    cache:
      host: localhost
      port: 6379
      password: ""
      database: 0
    bizcomp:
      host: localhost
      port: 6379
      password: ""
      database: 0

4. 애플리케이션 설정 ​

4.1 JWT 시크릿 키 설정 ​

src/main/resources/config/application-dev.yaml 파일에서 JWT 시크릿 키를 설정합니다.

yaml
jwt:
  secret: U29tZVN1cGVyU2VjcmV0S2V5Rm9ySlBXVC1NeVNlY3JldEtleQ==

주의: 운영 환경에서는 반드시 새로운 시크릿 키를 생성하여 사용하세요.

4.2 CORS 설정 ​

프론트엔드 도메인을 CORS 허용 목록에 추가합니다.

yaml
cors:
  allowed-origins: >
    http://localhost:8080
    ,http://localhost:5173
    ,http://localhost:3000
  mappings: >
    /api/**
    ,/auth/**

4.3 파일 경로 설정 ​

파일 업로드 경로를 설정합니다.

yaml
file:
  empimg:
    path: D:\\FileUpload\\HRMaster\\EMP\\
  report:
    path: D:\\FileUpload\\JasperReport\\
  path: D:\\FileUpload\\FCM\\COMFiles\\

주의: Windows와 Linux의 경로 형식이 다르므로 환경에 맞게 수정하세요.

5. 프로젝트 빌드 ​

5.1 Maven 의존성 다운로드 ​

bash
# 프로젝트 루트 디렉토리에서 실행
mvn clean install

# 또는 Maven Wrapper 사용
./mvnw clean install

5.2 빌드 확인 ​

빌드가 성공하면 target/ 디렉토리에 다음 파일이 생성됩니다:

  • AxpFramework-0.0.1-SNAPSHOT.war: 배포용 WAR 파일

6. 애플리케이션 실행 ​

6.1 IDE에서 실행 ​

IntelliJ IDEA ​

  1. 프로젝트를 IntelliJ IDEA로 열기
  2. AxpBootApplication.java 파일 열기
  3. main 메서드 옆의 실행 버튼 클릭
  4. 또는 Run → Run 'AxpBootApplication'

Eclipse ​

  1. 프로젝트를 Eclipse로 Import
  2. AxpBootApplication.java 파일에서 우클릭
  3. Run As → Java Application

6.2 Maven으로 실행 ​

bash
# 개발 환경으로 실행
mvn spring-boot:run -Pdev

# 또는 특정 프로파일 지정
mvn spring-boot:run -Plocal

6.3 WAR 파일로 실행 ​

bash
# 빌드
mvn clean package

# Tomcat에 배포
# target/AxpFramework-0.0.1-SNAPSHOT.war 파일을 Tomcat의 webapps 디렉토리에 복사

7.1 서버 시작 확인 ​

애플리케이션이 정상적으로 시작되면 다음과 같은 로그가 출력됩니다:

7.2 API 확인 ​

브라우저에서 다음 URL로 접속하여 확인합니다:

  • Swagger UI: http://localhost:8080/swagger-ui.html
  • Health Check: http://localhost:8080/api/health (구현된 경우)

7.3 로그 확인 ​

로그 파일은 다음 위치에 생성됩니다:

  • logs/AxpFramework/AxpFramework.log
  • logs/ess/ess-log.log
  • logs/fcm/fcm-log.log

8. 문제 해결 ​

8.1 포트 충돌 ​

8080 포트가 이미 사용 중인 경우:

yaml
# application-dev.yaml
server:
  port: 8081  # 다른 포트로 변경

8.2 데이터베이스 연결 실패 ​

  • 데이터베이스 서버가 실행 중인지 확인
  • 연결 정보(호스트, 포트, 사용자명, 비밀번호) 확인
  • 방화벽 설정 확인

8.3 Redis 연결 실패 ​

  • Redis 서버가 실행 중인지 확인
  • Redis 포트가 열려있는지 확인
  • Redis 비밀번호 설정 확인
