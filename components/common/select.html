선택 컴포넌트 ​

개요 ​

프로젝트에서 사용하는 선택(Select) 관련 컴포넌트들입니다. 다양한 형태의 선택 컴포넌트를 제공합니다.

SelectBoxSQL ​

개요 ​

SQL 쿼리 결과를 기반으로 하는 단일 선택 컴포넌트입니다. Vuetify의 v-combobox를 기반으로 구현되었습니다.

파일 위치 ​

src/views/ui/components/common/select/SelectBoxSQL.vue

Props ​

속성타입기본값설명
modelValueString-선택된 값 (v-model)
optionLabelString-표시할 라벨 필드명
optionValueString-값으로 사용할 필드명
defaultOptionBooleanfalse'선택' 옵션 추가 여부
readOnlyBooleanfalse읽기 전용 모드
labelString-라벨 텍스트
hiddenBooleanfalse선택된 값 숨김 여부
dataArray[]선택 옵션 데이터

사용 예제 ​

vue

<script setup>
    import {ref} from 'vue'

    const selectedValue = ref('')
    const options = ref([
        {code: 'D001', name: '개발팀'},
        {code: 'D002', name: '기획팀'},
        {code: 'D003', name: '디자인팀'},
    ])
</script>

<template>
    <p> selectedValue : {{ selectedValue }}</p>
    <v-row>
        <v-col cols="6">
            <SelectBoxSQL
                v-model="selectedValue"
                :data="options"
                option-label="name"
                option-value="code"
                default-option
                label="부서 선택"
            />
        </v-col>
    </v-row>
</template>

실제 컴포넌트 ​

selectedValue :

선택

API 연동 예제 ​

vue
<script setup>
import { ref, onMounted } from 'vue'
import axiosInstance from '@/CustomAxios.js'

const selectedValue = ref('')
const options = ref([])

const loadOptions = async () => {
  try {
    const response = await axiosInstance.post('/api/codes/departments')
    if (response.status === 'S') {
      options.value = response.data
    }
  } catch (error) {
    console.error('옵션 로드 실패:', error)
  }
}

onMounted(() => {
  loadOptions()
})
</script>

MultiSelectBoxSQL ​

개요 ​

다중 컬럼을 표시하는 선택 컴포넌트입니다. 테이블 형태로 옵션을 표시하며, 여러 컬럼 정보를 한 번에 볼 수 있습니다.

파일 위치 ​

src/views/ui/components/common/select/MultiSelectBoxSQL.vue

Props ​

속성타입기본값설명
modelValueString-선택된 값 (v-model)
optionLabelString-표시할 라벨 필드명
optionValueString-값으로 사용할 필드명
defaultOptionBooleanfalse'선택' 옵션 추가 여부
disabledBooleanfalse비활성화 여부
dropTypeString'bottom'드롭다운 위치 (bottom, top, center, start, end)
hiddenBooleanfalse선택된 값 숨김 여부
callbackFunction-선택 시 호출할 콜백 함수
columnsArray[]컬럼 정의 배열
dataArray[]선택 옵션 데이터

Columns 정의 ​

javascript
const columns = [
  { 
    caption: '코드',      // 컬럼 헤더 텍스트
    ref: 'code',          // 데이터 필드명
    width: '150px',       // 컬럼 너비
    style: 'text-align:left'  // 스타일
  },
  { 
    caption: '명칭', 
    ref: 'name', 
    width: '200px',
    style: 'text-align:left'
  },
]

사용 예제 ​

vue
<template>
  <MultiSelectBoxSQL
    v-model="selectedValue"
    :data="options"
    option-label="name"
    option-value="code"
    :columns="columns"
    default-option
    @update:modelValue="handleChange"
  />
</template>

<script setup>
import { ref } from 'vue'

const selectedValue = ref('')
const options = ref([
  { code: 'C001', name: '코드1', description: '설명1' },
  { code: 'C002', name: '코드2', description: '설명2' },
])

const columns = [
  { caption: '코드', ref: 'code', width: '150px', style: 'text-align:left' },
  { caption: '명칭', ref: 'name', width: '200px', style: 'text-align:left' },
  { caption: '설명', ref: 'description', width: '300px', style: 'text-align:left' },
]

const handleChange = (value) => {
  console.log('선택된 값:', value)
}
</script>

AG Grid에서 사용하기 ​

AG Grid의 셀 에디터로 사용할 수 있습니다.

GridMultiSelectWrapper ​

AG Grid에서 MultiSelectBoxSQL을 사용하기 위한 래퍼 컴포넌트입니다.

파일 위치: src/views/ui/components/grid/GridMultiSelectWrapper.vue

사용 예제 ​

vue
<script setup>
const columnDefs = ref([
  {
    field: 'department',
    headerName: '부서',
    editable: true,
    cellEditor: 'GridMultiSelectWrapper',
    cellEditorParams: {
      options: departmentOptions,
      columns: [
        { caption: '코드', ref: 'code', width: '100px' },
        { caption: '명칭', ref: 'name', width: '200px' },
      ],
      optionValue: 'code',
      optionLabel: 'name',
    },
  },
])
</script>

스타일 커스터마이징 ​

컴포넌트는 커스텀 CSS 클래스를 사용하므로 필요에 따라 스타일을 수정할 수 있습니다.

vue
<style scoped>
.custom-combobox {
  /* 커스텀 스타일 */
}
</style>

주의사항 ​

  1. 데이터 형식: data 배열의 각 객체는 optionLabel과 optionValue에 해당하는 필드를 포함해야 합니다.

  2. 반응성: data 배열이 변경되면 자동으로 옵션이 업데이트됩니다.

  3. 성능: 대량의 데이터를 사용하는 경우 가상 스크롤링을 고려하세요.
